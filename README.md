# 四色证据链观察器

小学语文课堂「四色证据链共学」的**教师端**观察工具原型。手机端优先，教师课堂中单手快速打点；前台极简，后台严谨。

> ⚠️ 本工具只做**教师端观察记录与已核验样本反馈**。不做学生端、不做个人排名、不做自动评分、不做全程录音；不保存真实姓名、人脸、声音。

---

## 四色含义（证据链四个环节）

| 色 | 名称 | 课堂环节 |
|---|---|---|
| ✏️ 铅笔 | 独立起点 | 学生独立初探 |
| 🖊️ 蓝笔 | 组内来源 | 组内交流、采纳同伴观点 |
| 🖋️ 黑笔 | 个人重构 | 个人再次组织、重构表达 |
| 🟥 红笔 | 全班修订 | 全班层面的修订与共识 |

打点时，四色代表课堂**当前环节**（时间序列），不是质量等级。

---

## 本轮产品语义：全员核验

课中小组按钮逻辑（v2，已从「组内出现」改为「全员核验」）：

1. **未点击小组 = 未核验（unknown）**，绝不参与判断，也不等于「未发生」。
2. **短按小组 = 教师已核验该组 N 名成员，N/N 均观察到当前目标证据。**
3. **长按小组 = 进入四人核验面板。**
4. 面板默认 N 人均「观察到」，教师**点掉暂未见**的成员。
5. 被点掉 = **「本次暂未见」**，绝不写成「不会/未完成/没做到」。
6. 点掉 4 号 → 该组显示 `3/4`。
7. 四人全点掉 → 结构允许 `0/4`，但文案仍是「暂未见」，不是失败评价。
8. **未核验小组不进入样本统计**。
9. 系统绝不输出「全班完成率/达成率/全员完成/多数学生已经完成/未完成」等误导词。
10. 反馈只基于「已核验样本」生成。

---

## 核心原则

1. 前台极简，后台严谨。
2. 手机端优先，教师课堂中只能单手快速点击。
3. 每节课默认只观察**一个主指标**，最多**一个副指标**。
4. **未核验 = unknown，绝不等于没有发生，也不进入统计。**
5. **短按 = 已核验该组全员；长按 = 进入逐人核验面板。**
6. 「暂未见」仅表示本次未观察到，不是「不会/未完成/没做到」。
7. 不做学生端、不做个人排名、不做自动评分、不做全程录音。
8. 不保存真实姓名、人脸、声音。
9. 反馈只能基于「已核验样本」生成，核验覆盖不足时不得形成全班判断。

## 功能页面

1. **班级小组设置页**：建班级（标签 + 组数 + 每组人数）。组与成员按位号自动命名（第 N 组 / 1~n 号），**无姓名输入**。
2. **课堂任务创建页**：选班级、课题、日期、主指标、**目标证据说明（可观察，会保存并回显）**、可选副指标、聚焦色阶。
3. **课中打点页**：顶部切「主/副指标」+ 四色色阶；中部 2 列大块小组网格。**短按** = 核验该组全员（N/N）；**长按** = 进入四人核验面板。
4. **四人核验面板**（长按触发，底部抽屉）：默认 N 人均观察到，点掉暂未见；底部「确认 / 撤销本组记录」。
5. **课后反馈页**：调用 `generateFeedback`，显示**核验覆盖**、已核验样本描述、说明与免责。

触控尺寸：返回 / 关闭 / 确认按钮 ≥44×44px；成员按钮 ≥56×48px；小组按钮单手友好。

## 数据模型（v2，全员核验）

```ts
type EvidenceStage = 'pencil' | 'blue' | 'black' | 'red';

interface Lesson {
  id; classId; title; date;
  primaryMetric: Metric;
  secondaryMetric?: Metric;
  focusStages?: EvidenceStage[];
  targetEvidence?: string;     // 目标证据说明（教师填写，可空）
}

interface ObservationFocus { lessonId; metricId; stage: EvidenceStage; }

// 核验型标记：一个 (观察点, 组) 至多一条
interface VerificationMark {
  focus: ObservationFocus;
  groupIndex: number;            // 1-based
  observedMembers: number[];     // 本次「观察到」的成员位号
  notObservedMembers: number[];  // 本次「暂未见」的成员位号（≠ 不会/未完成/没做到）
  verifiedAll: boolean;          // 是否 N/N
  timestamp: string;             // ISO
}

interface AppState { classes; lessons; verifications: VerificationMark[]; }
```

**没有任何 VerificationMark 的小组 = unknown（未核验），不进入统计分母，也不被当作 false。**

存储版本 `fccobserver` v2；v1（groupMarks/memberMarks，语义=组内出现）数据在加载时仅保留班级与任务配置，旧打点记录因语义不兼容被丢弃（控制台告警）。

## 状态管理

- React Context + `useReducer`（无额外依赖）。核验记录为 upsert（同槽位至多一条）/ 撤销。
- `localStorage` 形状校验 + 迁移；非法回退空状态，绝不从脏数据推断结论。
- 选择器 `src/selectors/observations.ts` 只产出四种状态：`unknown / verified_all / verified_partial / verified_none`。
- 反馈服务 `src/services/feedback.ts` 的 `generateFeedback(session, groups)` 为 mock，签名固定。

## 防误判保证

### A. 未核验 ≠ 未发生 / 不进入统计

- 存储：只有「已核验」记录进入统计；无记录 = unknown。
- 选择器：`unknown` 与 `verified_*` 严格区分；`unknown` 永不进入分母。
- UI：未核验小组用「中性灰 + 虚框 + 待观察」，绝不用红叉/空勾。
- 反馈：始终打印「已核验 M/N 组（共 X/Y 人）」；不输出「达成率/完成率」。

### B. 「暂未见」不是负面评价

- 数据：`notObservedMembers` 仅记录「本次暂未见」，语义非「不会/未完成/没做到」。
- UI：暂未见成员用**中性橙**，不是红叉；面板文案明确「不是不会/未完成/没做到」。
- 反馈：只说「X 人观察到、Y 人暂未见」。

### C. 反馈只基于已核验样本

- 覆盖率 = **已核验组占比**（核验覆盖），不是「出现证据占比」。
- 核验覆盖 < 50%（`COVERAGE_THRESHOLD`，`src/constants.ts`）→ 「核验样本不足，暂不形成全班判断」。
- 零样本 → 「尚无核验样本，不形成判断」。
- 永不出现「全班完成率/达成率/全员完成/多数学生已经完成/未完成」。

### D. 滚动不误打点

`useLongPress` 显式处理 `pointercancel`、移动超阈值、指针离开三种取消情形，统一置 `gestureCanceled`；`pointerup` 时若已取消或已长按，则**不**触发短按。滚动/平移由浏览器以 `pointercancel` 结束指针序列，已被显式拦截。

## API 与安全边界（重要）

1. **第一版不接真实 API**；`services/feedback.ts` 的 `generateFeedback` 是**纯本地 mock**，所有数据仅在本机 `localStorage`。
2. **未来接入模型 API（OpenAI / Claude / 其他）必须通过后端代理**，前端只与自有后端通信。
3. **严禁把任何模型 API key 写入前端代码**（`src/**`、`index.html`、配置文件等）。
4. **严禁使用 `VITE_*` 等环境变量向前端暴露真实 API key**——所有 `VITE_` 前缀变量都会被打包进前端产物，等同公开。
5. **不上传真实学生姓名、人脸、声音**；成员仅以位号引用。
6. **本地开发服务器监听局域网（`host:true`）只能在可信网络中使用**，切勿在公共/不可信 Wi-Fi 下开放。
7. **学生数据默认仅本地保存**；任何上传须显式获得授权并经后端代理。

## 依赖审计说明（npm audit）

`npm audit` 可能报告 esbuild 相关漏洞（开发依赖，来自 Vite）：

- 该告警**仅来自开发依赖**（devDependencies），**生产构建产物不包含 dev server**。
- `npm audit fix` 未能完全消除时，**不要为消除告警而引入复杂依赖或做不兼容的大版本升级**（与本工具目标相悖）。
- 缓解措施：
  - 不要把 `vite dev` / dev server 暴露到不可信局域网；
  - 生产对外服务应使用 `npm run build` 的产物（静态托管），**不使用 dev server**。

## 隐私

- 数据模型无姓名字段；UI 无姓名输入；成员仅以位号（1~n 号）引用。
- 全部数据仅保存在教师本机 `localStorage`，第一版无后端、不上传。
- 不录音、不拍照、不识别人脸。

## 已知限制（原型）

- 不跨设备同步（仅本机 localStorage）。
- 长按交互依赖 Pointer Events（现代浏览器均支持）。
- 反馈文案为规则模板（mock）；真实语义分析需经后端代理接入模型 API。
